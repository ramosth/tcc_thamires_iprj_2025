\chapter{OBJETIVOS}
\label{cap:objetivos}

% -------------------------------------------------------------
\section{Objetivo Geral}
\label{sec:objetivo-geral}
% -------------------------------------------------------------

Desenvolver um sistema automatizado de monitoramento e alerta para
barragens de rejeitos de mineração, baseado em tecnologia IoT, capaz
de mensurar continuamente a umidade do solo, integrar dados
meteorológicos obtidos por meio de APIs externas, calcular um índice
de risco geotécnico em tempo real e disponibilizar as informações em
uma plataforma \textit{web} de monitoramento, contribuindo para a
prevenção de acidentes em estruturas de contenção de rejeitos.

% -------------------------------------------------------------
\section{Objetivos Específicos}
\label{sec:objetivos-especificos}
% -------------------------------------------------------------

Para alcançar o objetivo geral, foram estabelecidos os seguintes
objetivos específicos:

\begin{enumerate}[label=\alph*)]

    \item Desenvolver um protótipo de \textit{hardware} embarcado
    baseado no microcontrolador ESP8266 NodeMCU, integrado a sensor
    higrômetro para medição contínua da umidade do solo, com
    sinalização visual por LEDs e sonora por \textit{buzzer}
    proporcional ao nível de risco identificado;

    \item Implementar a integração com as APIs meteorológicas
    BNDMET/DECEA e OpenWeatherMap para obtenção de dados de
    precipitação histórica acumulada nas janelas de 24 horas, 7 dias
    e 30 dias, e de previsão pluviométrica para as próximas 24 horas;

    \item Formular e implementar um modelo de avaliação de risco
    integrado composto por sete variáveis ponderadas, que expressam
    condições geotécnicas e meteorológicas, com mecanismo de
    amplificação ativado quando o solo apresenta saturação elevada
    concomitante a previsão de chuva moderada ou superior;

    \item Desenvolver o \textit{backend} do sistema com banco de dados
    de séries temporais, expondo uma API RESTful responsável pelo
    recebimento, processamento e persistência das leituras enviadas
    pelo dispositivo embarcado;

    \item Implementar uma interface \textit{web} de monitoramento em
    tempo real que permita visualizar o histórico de leituras, os
    níveis de alerta ativos e os dados meteorológicos integrados,
    além de gerenciar os usuários cadastrados para recebimento de
    notificações;

    \item Estabelecer um sistema de envio automático de alertas por
    e-mail aos usuários cadastrados quando o índice de risco atingir
    os níveis AMARELO (atenção) ou VERMELHO (crítico), acompanhado
    do relatório de dados do ciclo de análise correspondente.

\end{enumerate}
